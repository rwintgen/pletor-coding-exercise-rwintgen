### Final bonus pass — masonry, batch upload, dark mode
- **Masonry layout**: replaced fixed CSS grid with CSS multi-column (`column-count` 4→3→2→1 responsive). Cards now flow naturally and `break-inside: avoid` keeps them intact. Removed forced 4:3 aspect-ratio in `ImageCard` so each image displays at its real proportion. Shimmer placeholder still works via `min-height` until load.
- **Batch upload**: `DropZone` now accepts `multiple` files (drop or browse). `UploadForm` uploads sequentially, shows `Uploading 3/5…` progress, falls back to filename-as-title when shared title is empty, and pre-flight-checks the selection against remaining daily quota (refuses to start if it would exceed).
- **Dark mode (auto, follows system)**: introduced full CSS variable palette in `index.css` (`--neutral-0..950`, `--primary-50..700`, `--accent-400..600`, `--bg-page`, `--surface-glass`, `--scrollbar-thumb`, etc.). `theme.ts` `colors.neutral/primary/accent` now resolve to `var(--…)` so existing inline styles automatically swap. `@media (prefers-color-scheme: dark)` overrides every var with an inverted palette. Navbar glass and Auth shell radial backdrop use vars too. No flash, no toggle, no app-wide refactor.
